## P2-15: prd.yml 完整(pre-check 加 check-error-rate)

**Status**: 🟢 Done

**Goal**:把 P1-13 的 prd.yml pre-check composite 扩展为完整版,加入 `check-error-rate` 原子(Sentry Stats v2 API),错误率超阈值阻断部署。

**SPEC ref**:5.5 + 7.2

**Effort**:M(3 天)

**Dependencies**:P1-13(prd.yml 骨架已有)

**Files to create / modify**:
- `actions/prd/check-error-rate/action.yml`(原子,Sentry Stats v2 API + 阈值判定)
- `actions/prd/pre-check/action.yml`(P1-13 文件,加入 check-error-rate 串行调用)
- `tests/actions/check-error-rate.bats`(用 mock Sentry endpoint 验证 jq filter 与阈值)

**Constraints**(额外):
- 严格按 SPEC 7.2 节实现:
  - **Inputs**:`sentry-token`(必需)`sentry-org` `sentry-project` `environment`(default `staging`)`window-minutes`(default 30)`baseline-error-rate`(default 0.005)`threshold-multiplier`(default 2.0)`min-events`(default 100)
  - **Outputs**:`error-rate` `total-events` `passed`
- API endpoint **必须**是 `https://sentry.io/api/0/organizations/{org}/stats_v2/`(Stats v2,不是 v1)
- query string `field=sum(quantity)&groupBy=outcome&project={proj}&environment={env}&start={ts}&end={ts}&interval=1m`
- jq filter `outcome == "accepted"` 求和(过滤掉 dropped/rate_limited 噪声)
- 数据量门槛:`TOTAL < min-events` → skip 判定(`::notice` + `passed=true`,避免低流量项目误判)
- 阈值规则:`THRESHOLD = baseline × multiplier`;`RATE > THRESHOLD` → `::error title=Error Rate Exceeded` + exit 1
- bash 浮点用 `bc -l`,不用 awk(避免 locale 影响小数点)
- mock 测试时用 fixtures JSON(`tests/actions/fixtures/sentry-stats-*.json`),通过 `curl --offline` 或 `httpx` mock 替代真实调用

**Implementation outline**:
1. 写 `check-error-rate` action.yml,两个 step:`Query Sentry stats endpoint` + `Evaluate threshold`
2. step 1 用 `curl -sS -H "Authorization: Bearer $SENTRY_TOKEN"`,query 两次(全 outcome 和 category=error)
3. step 1 输出 `total` `errors` `rate` 三个 step output
4. step 2 读 step output + inputs 阈值,bc 比较
5. 把 check-error-rate 加到 `pre-check` composite,串行在 verify-version-align + observe-window 之后
6. bats 测试:fixture 高错误率 JSON → exit 1;低错误率 → exit 0;事件数低于 min → skip(exit 0 但 passed flag 仍为 true)

**Acceptance criteria**:
- [ ] e2e 仓库:配置真实 Sentry 测试 project,prd.yml 触发后 check-error-rate job 在日志中显示 `rate=X.XX threshold=Y.YY events=N`
- [ ] 故意制造 STG 环境高错误率(测试用 1% 阈值,manually 触发错误事件) → check-error-rate fail,prd 部署被阻断
- [ ] 错误率正常 → annotation `::notice title=Error Rate OK::rate=... ✓`
- [ ] STG 环境无流量(events < 100) → annotation `::notice title=Error Rate Check Skipped::events=X < min=100`,prd 继续
- [ ] mock test:`bats tests/actions/check-error-rate.bats` 用 fixtures 验证三种判定路径
- [ ] outputs.passed 在 prd.yml 中可被 `needs.pre-check.outputs.error-rate-passed` 读取(用于 annotation)
